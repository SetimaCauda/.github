# Guia de Contribuição

Obrigado por considerar contribuir com a Sétima Cauda.

Este documento reúne orientações simples para ajudar você a entender como participar do projeto de forma organizada, clara e respeitosa.

## Como contribuir

Você pode contribuir de várias formas:

* reportando bugs;
* sugerindo melhorias;
* corrigindo textos e documentação;
* enviando ajustes de interface;
* propondo melhorias de desempenho ou usabilidade.

## Antes de começar

Antes de abrir uma contribuição, recomendamos:

* verificar se o problema ou sugestão já foi discutido em uma issue;
* manter as mudanças pequenas e objetivas, sempre que possível;
* seguir o padrão de código e escrita já usado no projeto.

## Fluxo sugerido

1. Crie uma branch com um nome claro.
2. Faça suas alterações com foco em um objetivo por vez.
3. Teste localmente o que foi modificado.
4. Revise o conteúdo antes de enviar.
5. Abra um pull request explicando o que mudou e por quê.

## Padrões de escrita

* Use português brasileiro nos textos do projeto, quando aplicável.
* Mantenha frases claras e diretas.
* Evite jargões desnecessários.
* Prefira mensagens objetivas em commits, issues e pull requests.

## Padrões de código

* Preserve a estrutura e o estilo já adotados no projeto.
* Não remova comentários úteis sem necessidade.
* Sempre que adicionar comentários novos, escreva-os em português brasileiro.
* Faça alterações consistentes com o restante do código.

## Como abrir um pull request

Ao enviar sua contribuição, descreva:

* o que foi alterado;
* qual problema ou necessidade motivou a mudança;
* como testar, quando aplicável;
* qualquer observação importante para revisão.

## Boas práticas

* Seja respeitoso nas interações.
* Evite mudanças amplas sem alinhamento prévio.
* Priorize clareza, simplicidade e utilidade.
* Se algo estiver em dúvida, explique o contexto no pull request.

## Dúvidas

Se você não tiver certeza sobre como contribuir, abra uma issue ou entre em contato pelos canais oficiais do projeto.
